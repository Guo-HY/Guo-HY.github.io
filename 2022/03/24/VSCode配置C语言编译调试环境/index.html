    VSCode配置C语言编译调试环境

    发表于 2022-03-24
    使用vscode在windows环境下的windows C/C++编程与编译方法

    step0 一个vscode C项目的结构

    一个vscode项目会有一个.vscode文件夹在项目根目录下。这个文件夹中包含了对于该项目的vscode配置文件。具体到C/C++项目，创建后该文件夹中需要有两个文件（tasks.json，launch.json）才可以正常编译调试程序：

    • tasks.json:包含构建（编译）指令
    • launch.json:包含调试设置

    如果想有更加个性化的设置，这个文件必不可少:

    • c_cpp_properties.json：包含一些编译器设置与IntelliSense（用于自动代码补全，实时错误检查，代码改进建议）设置

    step1 配置tasks.json

    自动生成tasks.json文件的方法：在编辑器上方的终端选项中选择（应该是）最后一个：配置默认生成任务，然后在下拉菜单中选择构建所使用的编译器就可以了。

    我的tasks.json:

    • type条目指定了该以何种方式解释下面的command。如果值是shell，则当作命令行指令解析（bash，cmd，powershell等）。一般设置为shell就可以了。（默认生成的值是cppbuild，不太明白什么意思。）
    • command条目指定了编译器的绝对路径。
    • args条目指定了传递给编译器的参数，该条目中参数的位置必须按照编译器希望的顺序摆放。这里引用的一些变量可以参看附录1.通过改变这些变量，可以编译多个文件，或者使用一些其它的编译方式。具体怎么编译就看你对gcc命令的了解程度了。
    • label条目指定了你会在终端的tasks list中看到什么，可以随意设置。
    • “isDefault”: 设置为true表示使用Ctrl+Shift+B可以执行构建任务。

    执行构建工作

    使用Ctrl+Shift+B快捷键或者在编辑器上部的终端选项卡中选择运行生成任务。然后就有了.exe文件

    ![屏幕截图 2022-03-23 211641](https://raw.githubusercontent.com/Guo-HY/picture_git/master/img/屏幕截图 2022-03-23 211641.png)

    运行.exe文件：

    直接在终端中键入可执行文件名或者./可执行文件名。

    step2 配置launch.json

    自动生成launch.json文件的方法：在编辑器上方的运行选项卡中选择编辑配置...选项，然后可以根据需要选择一个，之后就会生成launch.json文件。

    生成的launch.json:

    • program - 指定了要调试的可执行文件。
    • args - 传递给程序的参数，使用双引号将参数括起来，参数间用逗号分隔。如果有转义字符，需要用双斜杠转义。
    • cwd - 指定了要调试的程序的工作目录。
    • stopAtEntry - 是否在main函数入口处停止。
    • MIMode：调制程序，只允许gdb或lldb。
    • externalConsole：是否为调试对象使用独立的控制台。
    • miDebuggerPath 调试程序路径
    • preLaunchTask - 指定了在执行运行或调试任务前要执行的任务。注意这里的值要和tasks.json中的label的值相同。

    开始调试

    使用F5或者Run > Start Debugging开始调试。

    在监视（WATCH）中点加号可以添加想要观察的变量。

    step3 编辑c_cpp_properties.json

    这个配置文件是可以可视化配置的：使用 Ctrl+Shift+P快捷键打开命令面板，然后选择**C/C++: Edit Configurations (UI)**就可以了。

    我的配置文件：

    • includePath：源文件随附的头文件目录，**表示递归搜索。如果在compilerPath中指定了编译器，就不用在这里设置系统include路径。
    • compilerPath：编译器位置

    附录 vscode配置文件中的变量

    在launch.json或tasks.json中，使用**${variableName}**这样的语法引用变量。

    预定义变量（部分）

    直接看例子吧。

    假设编辑器打开的文件的路径是：/home/your-username/your-project/folder/file.ext

    项目的根目录是/home/your-username/your-project

    • ${workspaceFolder} -/home/your-username/your-project

    • ${workspaceFolderBasename} - your-project

    • ${file} - /home/your-username/your-project/folder/file.ext

    • ${fileWorkspaceFolder} - /home/your-username/your-project

    • ${relativeFile} - folder/file.ext 这个是当前编辑文件相对于项目根目录的路径

    • ${relativeFileDirname} - folder

    • ${fileBasename} - file.ext

    • ${fileBasenameNoExtension} - file

    • ${fileDirname} - /home/your-username/your-project/folder

    • ${fileExtname} - .ext

    主要参考自vscode官方文档：https://code.visualstudio.com/docs/cpp/config-mingw
